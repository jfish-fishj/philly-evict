\section{Overview}\label{overview}

This note reports the streamlined annual transfer-evictions
specification. The main outcome is
\texttt{filing\_rate\ =\ num\_filings\ /\ total\_units} in a full
building-year panel from 2006 to 2019, with single-transfer treated
buildings compared against buildings never transferred in the RTT data.
All annual event studies are estimated on
\texttt{t\ \textbackslash{}in\ {[}r\ event\_window\_min,\ r\ event\_window\_max{]}},
with omitted event time \texttt{t\ =\ r\ event\_ref\_period}.

For filing outcomes, the main specification is a weighted stacked event
study:

\[
y_{i a t} = \sum_{k \in \{-3,-1,0,1,2,3\}} \beta_k \mathbf{1}\{\mathrm{rel\_time}_{ia}=k\}
          + \alpha_{ia} + \lambda_{at} + \varepsilon_{iat},
\]

estimated on a cohort-stacked panel with weights
\texttt{q\_weight\ ×\ total\_units}, where \texttt{q\_weight} is the
Wing et al.~corrective control weight and
\texttt{\textbackslash{}alpha\_\{ia\}} and
\texttt{\textbackslash{}lambda\_\{at\}} are stack-specific parcel and
year fixed effects. Non-filing outcomes continue to use the annual
\texttt{PID\ +\ year} event-study design on the same \texttt{-3:3}
window.

Operationally, \texttt{q\_weight} corrects the fact that the same
never-transferred controls are reused in multiple cohort stacks.
Multiplying by \texttt{total\_units} then makes the corrected stacked
estimator unit-weighted, so the filing results are a weighted average
over unit exposure rather than a simple parcel average.

The annual surface has been pruned to four blocks:

\begin{itemize}
\tightlist
\item
  filing-rate event studies
\item
  occupancy, resident-composition, complaint, and permit outcomes
\item
  rent outcomes using the unified and safe series
\item
  rental-evidence event studies for \texttt{1-unit} and
  \texttt{2+\ unit} buildings
\end{itemize}

Portfolio filing types are computed using the helper logic from the
production script, with portfolio size and filing rates defined using
all holdings, including \texttt{1-unit} buildings. The filing and
neighborhood-outcome regressions then restrict the estimation sample as
needed.

The acquirer bins are:

\begin{itemize}
\tightlist
\item
  \texttt{Single-purchase}: leave-one-out portfolio exposure is exactly
  \texttt{0} unit-years.
\item
  \texttt{Small\ portfolio}: leave-one-out portfolio exposure is
  positive but below 100 unit-years.
\item
  \texttt{Low-filer\ portfolio}: leave-one-out portfolio exposure is at
  least 100 unit-years and the leave-one-out filing rate is below 0.05
  filings per unit-year.
\item
  \texttt{High-filer\ portfolio}: leave-one-out portfolio exposure is at
  least 100 unit-years and the leave-one-out filing rate is at least
  0.05 filings per unit-year.
\end{itemize}

For the filing-rate outcome and for the leave-one-out filing-rate bins,
annual filing rates are capped at 0.75 filings per unit-year before
aggregation.

\section{Filing Heterogeneity by Acquirer
Type}\label{filing-heterogeneity-by-acquirer-type}

The main heterogeneity split uses four acquirer types:

\begin{itemize}
\tightlist
\item
  \texttt{High-filer\ portfolio}
\item
  \texttt{Low-filer\ portfolio}
\item
  \texttt{Small\ portfolio}
\item
  \texttt{Single-purchase}
\end{itemize}

These categories are defined from the helper-function leave-one-out
portfolio measures, while the regression itself is restricted to
\texttt{2+\ unit} buildings. In the current annual weighted stacked
outputs, the \texttt{t\ =\ +2} coefficient is 0.0325 for high-filer
portfolios and -0.0069 for low-filer portfolios.

The standard errors differ sharply across these bins. At
\texttt{t\ =\ +2}, the high-filer estimate has standard error 0.0436
versus 0.0068 for low-filer acquirers, which largely reflects the much
smaller treated sample for high-filer acquisitions.

\section{Rent Outcomes}\label{rent-outcomes}

The rent analysis is run on all units, but treated buildings are
required to have both pre- and post-acquisition support in the relevant
rent measure. The report keeps both the unified rent series and the safe
